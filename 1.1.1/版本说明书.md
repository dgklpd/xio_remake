# Xio 项目 1.1.1 版本说明书

## 1. 版本定位

- 版本号：`1.1.1`
- 版本性质：微信小程序前后端联调稳定版
- 目标平台：微信小程序真机横屏对战场景
- 本版重点：
  - 收口首页到战斗页的最小可玩流程
  - 强化 `BattlePage` 真机横屏适配稳定性
  - 将手牌交互改为“选中再确认”的移动端逻辑
  - 将小局播报改为轻量系统提示，并与小局比分同步

## 2. 本版核心更新

### 2.1 前端

- 首页保留单入口启动对局，并保留规则说明、更多内容两个占位入口。
- 战斗页完成横屏真机适配优化，重点处理：
  - 安全区与刘海屏边界
  - 敌我 HUD 固定角落布局
  - 中场战区真实测量定位
  - 底部手牌贴底与尺寸联动
- 手牌逻辑从“点击即出牌 / 长按易误触”改为“点选牌 -> 点场外确认出牌”的确认式流程。
- 小局结算播报改为系统 `toast`，同时装备栏闪烁 `+1`，不再使用占画面的自定义弹窗。
- 小局编号不再误用小局内部回合数，而是按比分推进逻辑显示“第 X 小局”。

### 2.2 后端

- WebSocket 单局对战接口稳定输出前端所需状态：
  - 手牌
  - 双方已出牌
  - 翻牌结算
  - 装备奖励
  - 比分
  - 倒计时
- 规则引擎继续以 `rules.py / basic_rules.py` 和 `backend/core/` 为核心来源，负责技能、装备、连锁、劣化、粉碎、死亡判定与装备发放。
- 后端状态中明确提供 `scores.my / scores.enemy`，供前端显示真正的小局比分。

## 3. 运行形态概览

### 3.1 用户侧流程

1. 进入首页。
2. 点击开始对局。
3. 打开战斗页并初始化 WebSocket。
4. 双方进入小局内回合流程：
   - 选牌
   - 锁牌
   - 翻牌
   - 结算
   - 小局胜负与装备奖励
5. 比分持续累计，直到整场结束。

### 3.2 当前页面结构

- 首页：`pages/index/index`
- 战斗页：`pages/battle/BattlePage`

## 8. 1.1.1 版本行为说明

### 8.1 战斗页交互

- 玩家点击一张手牌时，不会立即出牌。
- 被点击的牌会进入选中态。
- 再点手牌区空白，取消选中。
- 点手牌区外的战场空白，确认打出当前选中牌。

### 8.2 小局结算

- 每次小局分出胜负后：
  - 中央不再弹出大播报层
  - 直接使用系统 `toast` 提示
  - 若有装备奖励，对应装备栏闪 `+1`
- 小局文案格式：
  - `第 X 小局 我方获得 XXX`
  - `第 X 小局 对手获得 XXX`
  - `第 X 小局 我方获胜`
  - `第 X 小局 对手获胜`
  - `第 X 小局 平局`

### 8.3 真机适配

- 战斗页以横屏为唯一目标布局。
- 重点确保：
  - iPhone 系列真机安全区
  - HUD 不越界
  - 中央战场不漂移
  - 手牌贴底
  - 敌我区域不重叠
