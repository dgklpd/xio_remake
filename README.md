## 📝 项目简介

> [!NOTE]
> 
>​	“蓄”是我初中时期在学校里风靡的拍手游戏，该游戏因其快节奏和复杂的装备设计成为了在寄宿学校没有手机的我们平日里打发时间主要娱乐活动。初中那会刚接触计算机便痴迷于编程，梦想着有一天我能够将其做成游戏不用冒着被戒尺打的风险就来上一把蓄。
> 
>​	7年前，我用300行代码写出了一个python脚本向着这个理想迈出了第一步，全局变量维护状态，每个招式单独写函数，回合判定靠 if-else，电脑对手则是基于阈值和随机选择的简单逻辑……然而受限于当时的知识水平，我想要设计高水平的AI，想要设计实时的战斗界面，甚至只是想要复刻复杂的装备升级的机制都是不可能的事情，儿时的美梦终究被忙碌的学业冲淡，我的GitHub主页开始了长达6年的停更…… 

​	七年过去了，我变老了😋，也变强了，我学了无数不爱学的数学终于读上我心爱的计算机，如今的我拥有了丰富的理论知识和强大的AI工具，最重要的是我学习了体系化的软件工程开发思路，是时候再一次向这个难题发起挑战…… 	

---

### 🚀 全栈开发总览

仅仅四天，我利用 **Codex**, **Gemini**, **Claude** 核心工具完成了本项目的全栈开发，主要内容如下：

*   **规则算法优化**
    我设计了一套高鲁棒性的规则算法，通过多层联合判定，考虑技能复杂属性与粉碎情况的同时，通过面对对象的设计建模装备的成长曲线，实现该游戏核心规则建模，保证时间复杂度O（n）以快速判定。
*   **强化学习模型**
    我设计了一个用于非对称信息博弈的深度强化学习模型，该模型采用MaskablePPO算法，通过渐进式奖励函数驱动，在每局30回合的限制下，每小时可以与自己完成1000w自博弈对局，虽然目前的设计还达不到人类专家水准，但对战原有的随机模型已经可以达到97%的胜率。
*   **系统架构设计**
    我设计了一套前后端分离系统架构，前端采用UniApp + Vue 3 + TypeScript ，适配微信小程序，具备移动端响应式布局与丰富的动效。后段仍然采用python编写通过torch接入强化学习agent，基于 FastAPI 实现异步高并发处理，利用 WebSocket 监听战场实时变化。为解决实时对战并发问题，我引入 了SQLite 实现房间构建与玩家临时身份，真正把这个游戏从想法落地了下来。
