# Xio 项目 1.2.3 版本说明书

## 1. 版本定位
- 版本号：`1.2.3`
- 版本性质：联机房间制功能稳定版
- 目标平台：微信小程序（真机适配 + 联机对战）
- 本版重点：
  - 增加联机大厅与房间式对战流程
  - 统一等待/准备弹窗，直接叠加在战场之上
  - 联机战斗计时修正为 15 秒
  - 断线/退出即销毁房间（无用户库的最小化稳定方案）

## 2. 本版核心更新

### 2.1 前端
- 新增联机大厅页：
  - 房间列表以卡片气泡形式呈现。
  - 右上角加入/创建按钮统一动作。
  - 适配移动端安全区与顶部空间。
- BattlePage 联机模式：
  - 进入联机后不跳转新页面，等待弹窗直接悬浮在战场背景上。
  - 弹窗包含房间信息、玩家加入状态、准备/未准备状态气泡。
  - 底部并列“准备 / 退出”，随时退出并销毁房间。
  - 房间号显示 + 复制按钮，便于分享。
- 手牌交互逻辑优化：
  - 点击选中 → 再点空白确认出牌，避免误触。
- 联机用户最小化身份：
  - 使用 `guest` 身份并持久化本地 `guest_id`，减少重复创建导致的房间混乱。

### 2.2 后端
- 新增 `backend/online/` 联机模块：
  - 房间模型与 SQLite 存储。
  - 房间管理、Session 管理、双端 ready 流程。
  - WebSocket 状态同步（battle_state / room_state / round_resolved）。
  - 对战结束自动销毁房间。
  - 断线与退出即关闭房间（最小化稳定策略）。
- 战斗计时修正为 15 秒，并在双方出牌后取消旧定时任务，避免误触发。
- 支持 30 小局上限的对战流程控制。

## 3. 接口概览

### 3.1 HTTP
- `POST /online/rooms` 创建房间
- `POST /online/rooms/{room_code}/join` 加入房间
- `GET /online/rooms/{room_code}` 房间状态
- `POST /online/rooms/{room_code}/ready` 准备/取消准备
- `POST /online/rooms/{room_code}/leave` 退出并销毁房间

### 3.2 WebSocket
- `GET /ws/online/rooms/{room_code}?seat=host|guest&session_token=xxx`
- 主要事件：
  - `room_state`
  - `battle_state`
  - `round_resolved`
  - `room_closed`
  - `battle_finished`

- `backend/app/api.py`  
  FastAPI + WebSocket 入口
- `backend/online/`  
  房间与对战联机模块
- `backend/core/`  
  规则引擎与战斗逻辑

## 5. 已知限制
- 当前版本不含用户系统与长期身份绑定。
- 断线/退出策略为“立即销毁房间”，减少无用户库阶段的状态混乱。
